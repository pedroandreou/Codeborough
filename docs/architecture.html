Codeborough · System Architecture

A private, on-device, voice-first civic concierge for London. Four Docker containers on one box: who runs what, on which network, and what talks to what. The reasoning core has no internet route; the only door out is a default-deny proxy to ElevenLabs.

Networks on-device host edge_net · internet-facing core_net · internal, no internet off-device cloud
Built by reuse · OpenClaw NVIDIA · Nemotron our code & data infra · container
How it runs: the browser hits only bridge :8091. The reasoning core, vllm (the brain) and gateway (OpenClaw + civic-geo + memory + data), sits on core_net which is internal: true, so it has no route to the internet at all. The single way off the box is the default-deny egress-proxy, allowing only ElevenLabs. The brain is Nemotron, in vllm; OpenClaw (in the gateway) is the orchestrator that drives it and calls our civic-geo plugin, which answers from local London data.
Spine, every turn 1 browser to bridge :8091, the only inbound. 3 bridge to gateway over WebSocket + token. 4 gateway to vllm on a local /v1 API that uses the OpenAI API format (a common standard, so existing tools work) but runs entirely on the box: the brain reasons and tool-calls civic-geo over local data, then the answer returns to the gateway and bridge.
Voice only 2 before the brain reasons, bridge sends the spoken question out via the egress-proxy to ElevenLabs Scribe (speech to text); 5 afterwards it sends the answer to Eleven v3 (text to speech) so you hear it. A typed turn uses only steps 1, 3 and 4 and never touches ElevenLabs.
