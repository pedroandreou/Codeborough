A private, on-device, voice-first civic concierge for London. Four Docker containers on one box: who runs what, on which network, and what talks to what.

Docker container core_net (internal, no internet) edge_net (internet-facing) off-device (cloud) on-device host
How it runs: the user's browser hits only bridge :8091. The reasoning core - vllm (the brain) and gateway (OpenClaw + civic-geo + memory + data) - sits on core_net, which is internal: true, so it has no route to the internet at all. The only way out of the box is the default-deny egress-proxy, and it allows only ElevenLabs. Volumes (cb_models, cb_memory) are mounted into the containers that need them.
Request path 1 browser to bridge on :8091 (the only inbound) 2 bridge to gateway over WS :18789 with a token 3 gateway to vllm on the OpenAI /v1 API :8000 (the brain reasons + tool-calls civic-geo over the local GeoJSON) 4 for speech, bridge hands voice to the egress-proxy 5 which is the only thing allowed off the box, and only to ElevenLabs.
core_net is internal, so vllm and gateway have no internet route at all; cb_models is mounted into vllm and cb_memory + the GeoJSON datasets into gateway. Disable the optional walking-route and polling lookups and the box is fully air-gapped apart from voice.
