Codeborough · System Architecture

A private, on-device, voice-first civic concierge for London. Your location and queries never leave the device.

Reuse: OpenClaw runtime Nemotron: NVIDIA brain We write: civic-geo + data ElevenLabs: cloud voice On-device boundary Docker container (vllm / gateway) Hover to focus, click a box for detail
Voice turnThe user speaks and hears the answer on the device. Voice in, the brain calls civic-geo to look up live London data, then speaks a grounded answer back. Everything except ElevenLabs runs in Docker on the box: the brain (vllm) and the gateway (OpenClaw + civic-geo + memory + data) sit on an internal network with no route to the internet, so only the voice can leave, through the default-deny egress-proxy.

Voice turn 1 you speak (Scribe STT) 2 OpenClaw Gateway 3 Nemotron reasons 4 civic-geo tool call 5 local London GeoJSON lookup 6 the grounded answer returns through OpenClaw to Eleven v3 7 it is spoken back, you hear. Session memory saves and recalls every turn (over 1 h 11 m), a side loop rather than a numbered step.
Brain, data and memory are fully on-device; only the voice in and out goes through ElevenLabs, which that bounty requires.
