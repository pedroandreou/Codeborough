## What the data is `[commercial]`

Codeborough's civic facility data comes from London council open-data portals. Every record is published by a borough under the **UK Open Government Licence v3 (OGL)**, which permits redistribution, reuse, and adaptation with attribution. Sources and attribution per borough are documented in [`datasets/SOURCES.md`](../datasets/SOURCES.md).

The data is a static snapshot. It was downloaded and prepared for the June 2026 release; it does not update automatically. See [Refresh](#refresh) for how to update it.

## A simpler route for three facilities `[commercial]`

For three facility types, a single national dataset covers all of London:

| Facility | National source | Notes |
|---|---|---|
| **Schools** | DfE Get Information About Schools (GIAS) | UK-wide; already used as source for Camden's 658 rows |
| **Libraries** | Arts Council England | Already used for Camden; covers all English libraries |
| **Polling stations** | Democracy Club | UK-wide; changes per election |

Integrating these would immediately extend coverage for these three facilities to all 33 authorities. The borough-by-borough approach is unavoidable for **public toilets, grit bins, CCTV, and reception centres** — no national source exists.

## Data preparation pipeline `[developer]`

The following steps were performed to produce the files in `datasets/`. They were done out-of-band (one-off scripts, not committed). Only the prepared outputs are in the repository.

### Steps performed

**1. Format and projection unification**

All sources were converted to GeoJSON point geometry in WGS84 (longitude/latitude). Sources that published British National Grid eastings/northings (Wandsworth and Barnet CSVs) had their coordinates reprojected to WGS84.

*Residue in the data:* Camden library features still carry `Easting`/`Northing` fields in their properties, but the GeoJSON `geometry` coordinate is already lon/lat.

**2. Borough tagging**

Every feature was assigned a synthetic `_borough` property (e.g. `"camden"`, `"lambeth"`). This field does not exist in source data.

**3. Merging**

Per facility, all borough files were merged into one `<facility>-all-london.geojson`. The per-borough raw files remain in the repository alongside the merged file.

**4. Geocoding address-only sources**

Barnet schools and polling stations publish addresses but no coordinates. Coordinates were generated by geocoding to the postcode centroid via [postcodes.io](https://postcodes.io), so those points are accurate to postcode level, not surveyed positions.

**5. Source substitution**

Camden libraries were pulled from the national Arts Council England dataset (filtered to Camden), not Camden's own portal.

### What was NOT normalised on disk

Field names are kept verbatim from each borough's source portal. Schema unification (resolving `Establishment name` / `establishment_name` / `ESTABLISHMENT_NAME` to the same field) happens at read time in the `civic-geo` engine (`geo.mjs`). This keeps the raw data inspectable and avoids lossy renaming.

### Re-running the preparation

The prep scripts are not committed. To refresh a dataset:

1. Download the source data from the portal using the refresh URL (see [Refresh](#refresh) below).
2. Apply the preparation steps above (reproject if OSGB36, add `_borough` tag, merge into all-London file).
3. Run `node plugins/civic-geo/scripts/smoke.mjs` to validate.

## Caveats (read before relying on a layer) `[both]`

| Caveat | Detail |
|---|---|
| **Barnet schools + polling: postcode-level accuracy** | Coordinates are postcode-centroid geocodes via postcodes.io, not surveyed points. Accurate to ~50–100 m for well-populated postcodes. |
| **Lambeth CCTV = TfL traffic cameras** | Not council community-safety CCTV. Represents busy, well-served main roads. Do not frame as crime surveillance data. |
| **Camden schools = London-wide GIAS extract** | 658 rows all tagged `_borough: camden`, but individual rows carry their true `local_authority_name` (often another borough). Treat Camden schools as London-wide coverage until the file is re-split by authority. |
| **Kensington & Chelsea reception centres = emergency rest centres** | The nearest match to "reception centres" in K&C's data. Not routine customer-service desks. |
| **City of London polling = 4 places** | The City has a tiny resident population; 4 polling places is correct. City schools are catchment boundaries (polygons), not points, so they are not included. |
| **Schema heterogeneity preserved on disk** | Each borough's property field names are kept verbatim. The engine normalises them at read time — see [`plugins/civic-geo/src/geo.mjs`](../plugins/civic-geo/src/geo.mjs) `FIELDS` map for the mapping per facility. |
| **CCTV beyond Lambeth + K&C unavailable** | Most councils withhold camera locations. Barnet lists cameras but without coordinates. |
| **Reception centres beyond Camden + K&C unavailable** | Rare as open data. |

## Refresh `[developer]`

Download fresh source data from the portals using these URL patterns:

| Platform | URL pattern |
|---|---|
| **Socrata (Camden)** | `https://opendata.camden.gov.uk/resource/<ID>.geojson?$limit=50000` |
| **ArcGIS (Lambeth, Kingston, RBKC, H&F)** | `<FeatureServer or MapServer>/0/query?where=1=1&outFields=*&outSR=4326&f=geojson` |
| **WFS (City of London)** | `https://www.mapping.cityoflondon.gov.uk/arcgis/services/INSPIRE/MapServer/WFSServer?service=WFS&version=2.0.0&request=GetFeature&typeNames=<layer>&outputFormat=GEOJSON&srsName=EPSG:4326` |
| **Static CSV (Wandsworth, Barnet)** | Download the CSV and reproject OSGB36 eastings/northings to WGS84. |

After downloading:
1. Re-apply the preparation steps (reproject, `_borough` tag, merge).
2. Replace the relevant files in `datasets/`.
3. Run `node plugins/civic-geo/scripts/smoke.mjs` to validate.
4. Restart the gateway container (`docker compose restart gateway`) so the engine reloads.

The engine loads GeoJSON at startup via `readFileSync`. Changes to files are not hot-reloaded; a gateway restart is required.

## Why London's facility data is split across 33 portals `[commercial]`

London is governed at two levels: the GLA (Mayor, Transport for London, strategic policy) and 33 local authorities (councils, day-to-day services). Every facility in Codeborough is a **borough responsibility** — delivered and published independently by each council on its own data portal, in its own format.

There is no single "London facilities" database. The only exceptions are the three national datasets noted above (schools, libraries, polling stations). For all other facility types, data must be gathered borough by borough.

Full background: [`docs/london-structure.md`](london-structure.md).

## Licence `[commercial]`

All civic facility data in `datasets/` is released under the **UK Open Government Licence v3 (OGL)**. OGL permits:
- ✅ Copying, publishing, distributing, and transmitting the data
- ✅ Adapting the data
- ✅ Exploiting the data commercially and non-commercially

Conditions:
- You must acknowledge the source: *"Contains public sector information licensed under the Open Government Licence v3.0."*
- You must include a link to the licence: https://www.nationalarchives.gov.uk/doc/open-government-licence/version/3/

Per-source attribution (portal URLs and dataset identifiers) is in [`datasets/SOURCES.md`](../datasets/SOURCES.md).

> **Verify the licence on each portal before publishing.** OGL is standard for UK council open data but not universal. `datasets/SOURCES.md` notes this caveat. Source: [`datasets/SOURCES.md`](../datasets/SOURCES.md).
